# EduPlatform Student — Rust Native Mobile App

A native (non-WebView) student app built in **Rust + Slint**. True mobile feel with glassmorphism UI, compiled to APK and IPA via GitHub Actions.

Teachers and admins use the **web dashboard** at the repository root (Next.js). This folder is **students only**.

## Stack
- **UI**: [Slint 1.7](https://slint.dev) — declarative Rust UI, native rendering, custom shaders
- **HTTP**: `reqwest` + `rustls-tls`
- **Async**: `tokio` multi-thread runtime
- **Storage**: OS keyring (`keyring` crate) for the session token
- **Build**: `cargo-mobile2` for Android + iOS scaffolding
- **CI**: GitHub Actions (`.github/workflows/build-android-and-ios.yml`)

## Glassmorphism design system
- Deep navy background gradient (`#0B1120 → #131A2E → #1E293B`)
- Frosted glass cards: `rgba(255,255,255,0.06)` with subtle border glow
- Emerald accent: `#10B981`
- Soft drop shadows (blur 18–24px, offset 6–10px)
- Rounded corners 14–28px
- Bottom nav floats with glass blur, no system bar
- No "AI slop" gradients or rainbow buttons — minimal, modern, intentional

## Develop locally

### Prerequisites
- Rust 1.80+ (`rustup default stable`)
- Linux: `sudo apt install libxcb1-dev libxkbcommon-dev libgles2-mesa-dev libfreetype6-dev libfontconfig1-dev`
- macOS: Xcode Command Line Tools (`xcode-select --install`)
- The backend API running (the Next.js app at the repo root, or a deployed URL)

### Build & run on desktop
```bash
cd student-app-rust
export EDUPLATFORM_API_URL=http://localhost:3000
cargo run --release
```
A 390×844 window opens (iPhone 14 logical size) for testing the UI.

### Build for Android (one-time setup)
```bash
# Install Android targets
rustup target add aarch64-linux-android armv7-linux-androideabi x86_64-linux-android i686-linux-android

# Install cargo-mobile2
cargo install --git https://github.com/BrainiumLLC/cargo-mobile2 --locked

# Generate the Android project
cargo mobile init --platform android --name eduplatform-student --package app.eduplatform.student

# Build debug APK
cargo mobile build --platform android --profile debug --target aarch64
```

### Build for iOS (macOS only)
```bash
rustup target add aarch64-apple-ios aarch64-apple-ios-sim
cargo mobile init --platform ios --name eduplatform-student --package app.eduplatform.student
cargo mobile build --platform ios --profile release --target aarch64
```

## GitHub Actions CI

The workflow at `.github/workflows/build-android-and-ios.yml` triggers on every push to `main` and on every `v*` tag.

### On push to main
- ✅ Builds debug APK → uploaded as artifact
- ✅ Runs lint + tests

### On `v*` tag (release)
- ✅ Builds release APK + AAB → uploaded as artifacts
- ✅ Publishes AAB to Google Play (internal track)
- ✅ Builds iOS IPA → uploaded as artifact
- ✅ Publishes IPA to App Store Connect via `xcrun altool`

### Required GitHub Secrets

Go to **Settings → Secrets and variables → Actions** and add:

| Secret name | Description |
|-------------|-------------|
| `EDUPLATFORM_API_URL` | Backend API base URL, e.g. `https://api.eduplatform.app` |
| `GROQ_API_KEY` | Groq API key for AI question generation |
| `EDUPLATFORM_KEYSTORE_BASE64` | `base64 -w0 eduplatform.jks` output |
| `EDUPLATFORM_KEYSTORE_PASSWORD` | Keystore password (also alias password) |
| `PLAY_STORE_SERVICE_ACCOUNT_JSON` | Google Play service account JSON (plain text) |
| `APPLE_DEVELOPMENT_TEAM` | Apple Developer Team ID |
| `APPLE_PROVISIONING_PROFILE` | Provisioning profile name |
| `APPLE_CODE_SIGN_IDENTITY` | Usually "iPhone Distribution" |
| `APP_STORE_CONNECT_API_KEY` | base64-encoded `.p8` key file |
| `APP_STORE_CONNECT_API_KEY_ID` | Key ID from App Store Connect |
| `APP_STORE_CONNECT_ISSUER_ID` | Issuer ID from App Store Connect |

### Generate the keystore (one-time)
```bash
keytool -genkeypair -v \
  -keystore eduplatform.jks \
  -alias eduplatform \
  -keyalg RSA -keysize 2048 \
  -validity 10000

# Encode for GitHub Secrets
base64 -w0 eduplatform.jks
# Copy the output → EDUPLATFORM_KEYSTORE_BASE64
```

## Backend API contract

The student app talks to the Next.js backend. All endpoints expect `Authorization: Bearer <token>` (returned by `/api/auth/verify-otp`).

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/auth/request-otp` | POST | `{ contact }` → sends OTP |
| `/api/auth/verify-otp` | POST | `{ contact, code, role }` → returns `{ token, user }` |
| `/api/auth/me` | GET | Returns current user |
| `/api/auth/logout` | POST | Invalidates session |
| `/api/student/home` | GET | Home dashboard stats |
| `/api/student/subjects` | GET | List subjects |
| `/api/student/subjects/:id/chapters` | GET | List chapters in subject |
| `/api/student/chapters/:id/lessons` | GET | List lessons in chapter |
| `/api/student/tests` | GET | List available tests |
| `/api/student/tests/:id` | GET | Full test with questions |
| `/api/student/tests/:id/submit` | POST | Submit answers → score |
| `/api/student/qa` | GET / POST | List / post Q&A questions |

## Security
- Session token stored in OS keyring (Keystore on Android, Keychain on iOS, libsecret on Linux)
- All network traffic over HTTPS (TLS 1.2+ via `rustls-tls`)
- No tracking, no analytics, no third-party SDKs
- CSP and HSTS enforced by the backend
